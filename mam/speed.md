# Скорость

Сравнение со старым сборщиком: холодная CLI-сборка, перед каждым прогоном
удалены все сгенерированные артефакты (`-`, `-mam`, `-web`, `-node`).
Замер 2026-07-21, macOS (arm64), Node 24.

| Цель | Старый (`npx mam`) | Новый (`node mam/start/-/node.js`) |
|---|---|---|
| hyoo/harp/app | 15.1s | **6.0s** |
| mol/app/docs | 62.6s | **12.8s** |

Основной выигрыш — тайпчекер работает в worker-треде параллельно сборке:
загрузка CPU у нового 170–185% против 46–78% у старого. Аудит-бандлы
при этом генерируются, сравнение честное.

Тёплый повторный прогон у обоих равен холодному — CLI-процесс
межзапускного кеша не имеет. Инкрементальность работает в серверном
режиме: пересборка по вотчеру переиспользует реактивный граф в памяти.
